Boas Práticas — Desenvolvimento com IA

Com IA / Agentes (Claude & Gemini)

  • Evite o "Vibe Coding": Não peça para a IA codar logo de cara. Exija o fluxo: Explore (leia arquivos) → Planeje (proponha solução) → Implemente.
  • O Padrão "Documentar e Limpar": Sessões longas fazem a IA alucinar. Peça para a IA resumir o progresso num arquivo .md, use /clear (zere a memória) e retome a partir do documento.
  • Revisão Cruzada (Multi-Agente): A mesma sessão que escreveu o código é ruim para revisá-lo. Abra um terminal fresco para revisar o PR de forma crítica.
  • CLAUDE.md: Mantenha-o enxuto (max 200 linhas). Documente nele os erros que a IA costuma repetir no seu projeto.

Com Snowflake + Python

  • Nunca armazene credenciais no código — use .env
  • Adicione .env ao .gitignore
  • Use queries parametrizadas (evite SQL injection)
  • Feche conexões com conn.close() ou with
  • Suspenda o warehouse após análises longas
  • Prefira pd.read_sql() para resultados em DataFrame
Fluxo recomendado: Prototipar no Jupyter → Revisar com Claude Code → Refatorar em script → Agendar no pipeline de dados da empresa.
