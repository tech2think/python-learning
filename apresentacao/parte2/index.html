O que é Claude Code?

Claude Code é uma ferramenta de linha de comando da Anthropic que integra o modelo Claude diretamente ao terminal e ao seu editor de código.
Ele lê e escreve arquivos, executa comandos, entende o contexto do seu projeto e colabora como um desenvolvedor experiente — mas você mantém o controle total.
Diferente de autocompletar de IDE: Claude Code entende o projeto inteiro e pode refatorar, documentar e gerar testes de forma contextual.

Casos de uso para analistas de energia

  • Gerar queries SQL para Snowflake a partir de descrições em português
  • Criar funções de validação de dados de UC
  • Documentar pipelines de qualidade automaticamente
  • Explicar erros e sugerir correções
  • Refatorar notebooks em scripts produtivos
  • Gerar relatórios de qualidade em Excel/PDF

M4 Instalação e Configuração

Pré-requisitos

  • Node.js 18+ instalado
  • Conta na Anthropic (anthropic.com)
  • Chave de API (API Key)

Instalação

# Instalar globalmente via npm
npm install -g @anthropic-ai/claude-code

# Configurar a API Key
export ANTHROPIC_API_KEY="sk-ant-..."

# Ou adicionar ao ~/.bashrc / ~/.zshrc
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' \
  >> ~/.bashrc

Primeiros comandos

# Iniciar Claude Code no projeto
cd /meu/projeto
claude

# Modo não-interativo (comando único)
claude -p "Explique este arquivo"

# Verificar versão
claude --version
Ao iniciar o Claude Code em um projeto com CLAUDE.md, ele carrega automaticamente o contexto do projeto.

M4 Estrutura de Projetos com CLAUDE.md

O que é o CLAUDE.md?

É um arquivo Markdown na raiz do projeto que fornece contexto persistente ao Claude Code: quem você é, o que o projeto faz, quais padrões seguir e o que não fazer.

O CLAUDE.md é lido a cada sessão. Mantenha-o atualizado — é a memória do seu projeto.

Seções recomendadas

  • Contexto do negócio e domínio
  • Stack tecnológica e versões
  • Convenções de nomenclatura
  • Padrões de código e estilo
  • O que Claude NUNCA deve fazer
  • Comandos úteis do projeto

Exemplo — qualidade de dados

# Projeto: Qualidade Cadastro UC

## Contexto
Pipeline de qualidade do cadastro de
consumidores de uma distribuidora de
energia. Dados no Snowflake.

## Stack
- Python 3.11, pandas, great-expectations
- Snowflake (schema: CADASTRO)
- Tabela principal: CONSUMIDORES_UC

## Convenções
- Nomes em snake_case
- Prefixo vw_ para views
- Queries parametrizadas (sem f-strings
  em SQL para evitar SQL injection)

## Nunca
- Fazer DELETE/UPDATE sem WHERE
- Commitar credenciais no .env

M4 Skills e Prompts Customizados

O que são Skills?

Skills são arquivos Markdown que definem tarefas reutilizáveis para o Claude Code. Você invoca com /nomeskill e ele executa a tarefa descrita.

# .claude/commands/data-quality.md
---
description: Executa análise de qualidade
  no DataFrame `df`
---

Analise a qualidade dos dados no
DataFrame `df`:
1. % nulos por coluna
2. Duplicatas na chave primária
3. Outliers em colunas numéricas
4. Gere um relatório em Markdown
Salve skills em .claude/commands/ no projeto.

Prompt engineering para dados

✅ Prompt eficaz:

"Crie uma função Python que recebe um
DataFrame pandas com a tabela CONSUMIDORES_UC
(colunas: cod_consumidor, cpf_cnpj,
cod_modalidade_tarifaria, vlr_consumo_medio_kwh)
e retorna um dict com:
- pct_nulos: % nulos por coluna
- duplicatas: contagem de cod_consumidor
  duplicados
- outliers_consumo: registros com consumo
  acima de 3 desvios padrão
Use type hints e docstring em português."
❌ Prompt ineficaz:
"faça uma função de qualidade de dados"

M5 Snowflake + Python: Arquitetura

Componentes Snowflake

AccountInstância da sua empresa
WarehouseCluster de computação (VXS a 6XL)
DatabaseBanco de dados lógico
SchemaAgrupamento de tabelas/views
StageÁrea de carga de arquivos
Snowflake cobra por tempo de execução do warehouse. Sempre suspenda ao terminar análises longas.

M5 Queries Avançadas no Snowflake

CTEs — organização de queries

query = """
WITH nulos_por_campo AS (
  SELECT
    COUNT(*) AS total,
    COUNT(cpf_cnpj) AS com_cpf,
    COUNT(num_medidor) AS com_medidor,
    COUNT(num_cep) AS com_cep
  FROM CONSUMIDORES_UC
  WHERE flg_ativo = TRUE
),
pct_preenchimento AS (
  SELECT
    ROUND(com_cpf / total * 100, 1)
      AS pct_cpf,
    ROUND(com_medidor / total * 100, 1)
      AS pct_medidor
  FROM nulos_por_campo
)
SELECT * FROM pct_preenchimento;
"""
df = pd.read_sql(query, conn)

Window Functions — ranking de issues

query_rank = """
SELECT
  nom_municipio,
  COUNT(*) AS total_ucs,
  SUM(CASE WHEN cpf_cnpj IS NULL
    THEN 1 ELSE 0 END) AS sem_cpf,
  ROUND(
    SUM(CASE WHEN cpf_cnpj IS NULL
      THEN 1 ELSE 0 END)
    / COUNT(*) * 100, 1
  ) AS pct_sem_cpf,
  RANK() OVER (
    ORDER BY pct_sem_cpf DESC
  ) AS ranking
FROM CONSUMIDORES_UC
GROUP BY nom_municipio
ORDER BY ranking
LIMIT 20;

Boas Práticas — Desenvolvimento com IA

Com Claude Code

  • Mantenha o CLAUDE.md atualizado com o contexto do projeto
  • Revise todo código gerado antes de executar
  • Use o Claude para explicar, não apenas para gerar
  • Versione os prompts e skills junto com o código
  • Documente o que o Claude gerou nos commits

Com Snowflake + Python

  • Nunca armazene credenciais no código — use .env
  • Adicione .env ao .gitignore
  • Use queries parametrizadas (evite SQL injection)
  • Feche conexões com conn.close() ou with
  • Suspenda o warehouse após análises longas
  • Prefira pd.read_sql() para resultados em DataFrame
Fluxo recomendado: Prototipar no Jupyter → Revisar com Claude Code → Refatorar em script → Agendar no pipeline de dados da empresa.
