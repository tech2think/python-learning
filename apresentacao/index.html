Construindo a base sólida necessária para automação e análise de dados no setor elétrico.

M1 Variáveis e Tipos Básicos

Tipos Numéricos

# Inteiros (int) — contagens, códigos
cod_consumidor = 100042
tensao_nominal = 220 # Volts

# Ponto Flutuante (float) — medições, valores
consumo_kwh = 342.7
fator_potencia = 0.92

Textos e Lógicos

# Textos (str) — nomes, categorias
nome = "Maria Silva Santos"
municipio = "São Paulo"

# Booleanos (bool) — verdadeiro/falso
ativo = True
inadimplente = False

Regras Importantes

  • Tipagem dinâmica: não precisa especificar o tipo na criação.
  • Nomes não podem ter espaços ou caracteres especiais (exceto _).
  • Case Sensitive: a variável X é diferente da variável x.
  • Proibido usar palavras reservadas (if, def, True, etc.) como nomes.
  • Use # para inserir comentários no seu código.

M1 Estruturas de Dados

Listas (Sequências)

Coleções ordenadas e mutáveis de itens.

# Criando uma lista de leituras
leituras = [310, 325, 342, 298, 315]

# Acessando o primeiro elemento (índice 0)
primeira_leitura = leituras[0]

# Adicionando um novo elemento
leituras.append(330)

Dicionários (Chave-Valor)

Coleções de pares com chaves únicas.

# Criando um registro de UC
consumidor = {
  "cod": 100042,
  "nome": "Maria Silva",
  "classe": "B1",
  "consumo_kwh": 342.7,
  "ativo": True
}

# Acessando valores
nome_cliente = consumidor["nome"]

Regras e Dicas

  • Listas: o primeiro item está sempre no índice 0.
  • Use len(lista) para saber o número de itens.
  • Listas preservam a ordem em que os itens foram inseridos.
  • Dicionários: as chaves (ex: "nome") devem ser únicas.
  • Acesso rápido: encontrar um valor pela chave é instantâneo.
  • Ideal para representar objetos complexos (clientes, medidores).

M1 Estruturas de Controle

Classificação tarifária (if/elif)

def classificar_tarifa(consumo_kwh):
  if consumo_kwh <= 30:
    return "Baixa Renda"
  elif consumo_kwh <= 100:
    return "Faixa 1"
  elif consumo_kwh <= 220:
    return "Faixa 2"
  else:
    return "Faixa 3"

print(classificar_tarifa(342)) # → "Faixa 3"

Iteração sobre leituras (for)

leituras = [310, 325, 342, None, 315]

validas = []
for leitura in leituras:
  if leitura is not None:
    validas.append(leitura)

# List comprehension — mais pythonico
validas = [x for x in leituras
           if x is not None]

media = sum(validas) / len(validas)
print(f"Média: {media:.1f} kWh")

Regras e Dicas

  • Indentação é lei: Em Python, os blocos de código são definidos pelos espaços no início da linha (use 4 espaços).
  • Não esqueça: É obrigatório usar dois-pontos (:) no final das linhas de if, elif, else e for.
  • if/elif/else: Em uma cadeia de condições, apenas o bloco da primeira condição verdadeira será executado.
  • for loop: É a forma mais direta e pythônica de percorrer itens de uma lista um por um.

M1 Funções

Cálculo do DEC

def calcular_dec(interrupcoes: list) -> float:
  """
  Calcula o DEC (Duração Equivalente de
  Interrupção por Consumidor).

  Args:
    interrupcoes: lista de dicts com
      'duracao_h' e 'consumidores'
  Returns:
    DEC em horas
  """
  total_ucs = interrupcoes[0]['total_ucs']
  soma = sum(
    i['duracao_h'] * i['consumidores']
    for i in interrupcoes
  )
  return round(soma / total_ucs, 4)

Validação de CPF

import re

def validar_cpf_formato(cpf: str) -> bool:
  """Valida formato do CPF."""
  cpf_limpo = re.sub(r'\D', '', cpf)
  return len(cpf_limpo) == 11

def formatar_cpf(cpf: str) -> str:
  """Formata CPF: 12345678900 →
  123.456.789-00"""
  c = re.sub(r'\D', '', cpf)
  if len(c) != 11:
    return cpf
  return f"{c[:3]}.{c[3:6]}.{c[6:9]}-{c[9:]}"

Regras e Dicas

  • Reutilização: Use funções para evitar repetir lógica de cálculo (como DEC/FEC) em vários lugares.
  • Docstrings: Sempre documente o que a função faz, seus parâmetros e o que ela retorna entre """triple quotes""".
  • Type Hints: Indicar tipos (ex: cpf: str) ajuda na leitura do código e evita bugs.
  • Modularidade: Mantenha funções focadas em uma única tarefa para facilitar testes e manutenção.

M2 Limpeza de Dados de Consumidores

Problemas comuns no cadastro

  • CPF/CNPJ com formatação inconsistente
  • Nomes em maiúsculas, minúsculas e misto
  • CEP sem zeros à esquerda
  • Duplicatas de UC por migração de sistema
  • Datas em formatos diferentes
  • Campos obrigatórios vazios
  • Coordenadas geográficas ausentes
Exemplos de Limpeza: Padronização de nomes de consumidores, saneamento de documentos (CPF/CNPJ) e remoção de registros duplicados em grandes bases cadastrais.

Técnicas de limpeza

# Normalizar nomes
df['nom_consumidor'] = (
  df['nom_consumidor']
  .str.strip()
  .str.title()
)

# Limpar CPF/CNPJ
df['cpf_cnpj'] = (
  df['cpf_cnpj']
  .str.replace(r'\D', '', regex=True)
)

# Remover duplicatas
df = df.drop_duplicates(
  subset=['cod_consumidor']
)

Regras e Dicas

  • Padronização: O uso de .str.title() garante que "MARIA", "maria" e "Maria" sejam tratadas como a mesma pessoa ("Maria").
  • Regex: Expressões regulares (r'\D') são poderosas para extrair apenas os números de CPFs mal formatados.
  • Duplicatas: Ao remover duplicatas (drop_duplicates), defina qual coluna deve ser única com o parâmetro subset.
  • Impacto: Lembre que dados de clientes limpos são cruciais para cobrança correta e relatórios regulatórios.

M2 Análise de Qualidade do Cadastro

Score de qualidade por registro

def score_qualidade(row) -> float:
  campos = [
    'nom_consumidor', 'cpf_cnpj',
    'cod_modalidade_tarifaria',
    'nom_municipio', 'num_cep',
    'num_medidor', 'dat_ultima_leitura'
  ]
  preenchidos = sum(
    1 for c in campos
    if pd.notna(row[c]) and row[c] != ''
  )
  return round(preenchidos / len(campos), 2)

df['score_qualidade'] = df.apply(
  score_qualidade, axis=1
)

Dimensões de qualidade

# Completude (%)
completude = (
  df['nom_consumidor'].notna().mean() * 100
)

# Unicidade (Duplicatas)
duplicados = (
  df.duplicated(subset=['cod_consumidor']).sum()
)

# Consistência
consistente = (
  df['cod_classe_consumo'].isin(['RES', 'IND', 'COM'])
)

Regras e Dicas

  • Completude: Verifique se campos críticos (CPF, Medidor) estão preenchidos para evitar erros de faturamento.
  • Unicidade: O cod_consumidor deve ser único para garantir a integridade da base.
  • Consistência: Cruze dados (ex: Classe vs. Tarifa) para validar se as regras de negócio foram aplicadas.
  • Score: Use o .apply(axis=1) para calcular indicadores linha a linha de forma simples.

M3 Indicadores DEC e FEC

DEC — Duração Equivalente de Interrupção

Soma das durações de interrupções ponderadas pelo número de UCs afetadas, dividida pelo total de UCs do conjunto.

# Fórmula
DEC = Σ(Duração_i × UCs_i) / UCs_total

FEC — Frequência Equivalente de Interrupção

Número de interrupções ponderadas pelo número de UCs afetadas, dividido pelo total de UCs.

FEC = Σ(UCs_i) / UCs_total
import plotly.graph_objects as go

Regras e Dicas

  • Cálculo Crítico: DEC e FEC são os indicadores de qualidade de fornecimento mais observados pela ANEEL e pela diretoria.
  • Ponderação: Lembre que uma interrupção de 1h para 1000 clientes é mais grave no DEC do que 2h para 10 clientes.
  • Visualização: Sobrepor o realizado (barra) com a meta regulatória (linha tracejada) é a forma padrão do setor de apresentar o DEC.
  • Qualidade: Erros no cadastro de UCs (tema da aula anterior) afetam diretamente o denominador do cálculo!
